option_almanac module¶

General-purpose utilities for meep.adjoint

This file collects some general-purpose utility functions and classes that are used in meep.adjoint but are not specific to adjoint solvers or computational electromagnetism.

The functions/classes included are as follows:

  1. process_options: creates {option:value} dicts by parsing a

    hierarchy of config files, environment variables, and command-line options

  2. math utility routines

class meep_adjoint.option_almanac.OptionAlmanac(templates, custom_defaults={}, section=None, filename=None, search_env=True, prepend_section=False)[source]¶

Bases: object

Database of option values read from config files, environment, and command-line args.

On instantiation, this class determines values for each element in a list of named options by consulting each of the following sources in increasing order of priority:

  1. The default value given in the initialization template (see below)

  2. Dict of customized defaults passed to constructor or set_defaults()

  3. Global config file

  4. Local config file

  5. Environment variables

  6. Command-line arguments. (Processed arguments are removed from sys.argv, leaving other arguments unchanged for subsequent downstream parsing.)

After initialization, class instances serve as databases of option values that may be queried via the __call__ method, thus:

my_options = OptionAlmanac(…) opt_value = my_options(‘option_name’)

Constructor arguments:

templates (list of OptionTemplate): option definitions

custom_defaults (dict): optional overrides of default values in templates

section(str): optional section specification; if present, only matching

sections of config files are parsed. If absent, all sections of all files are parsed.

filename (str): If e.g. filename == ‘myconfig.rc’, then:

  1. the global config file is ~/.my_config.rc

  2. the local config file is ./my_config.rc in the current working directory.

If unspecified, no configuration files are processed.

search_env (bool): Whether or not to look for option settings in

environment variables. The default is True, but you may want to set it to False if you have option whose names overlap with standard environment variables, such as ‘HOME’

prepend_section (bool): If True, the section name (plus an underscore) is

prepended to the names of options for searching environment variables and command-line arguments.

enforce_type(name, value)[source]¶

Typechecking/conversion for option update candidates :param name: name of an option :type name: str :param value: a value somebody has proposed to assign the option :type value: ?

Returns: value converted to the type of the given option, or None

if this is not possible. The conversion is attempted by a simple typecast unless the return type is boolean, which we process by hand as automatic conversion does not work [i.e. bool(‘False’) == True].

revise(revisions, context)[source]¶

Cautiously apply a proposed set of candidates for updated option values. :param revisions: dict of {key:new_value} records OR list of (key,new_value) pairs :param context: optional str like ‘global config file’ or ‘command line’

for inclusion in error messages to help trace mishaps

Parameters

  • result is similar to that of self.options.update( {k (The) – v for (k,v) in revisions } ),

  • but (a) –

    1. removes non-escaped single and double quotes surrounding strings,

    2. attempts type conversions as necessary to preserve the type of the value associated with each key.

Return value: None (the class-internal dict of option settings is updated in-place)
