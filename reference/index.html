General reference¶

This page documents various conventions and protocols adopted by meep_adjoint.

Tweaks to meep conventions¶

The meep_adjoint package introduces a few minor updates or revisions to some of the standard ways of doing business in meep and the meep python module .

Unified handling and labeling of spatial regions: The Subregion class:¶

A historical idiosyncrasy of the meep C++ back-end engine, which was retained in the definition of the meep python interface , is the use of multiple (at least seven 1) distinct data structures to describe subregions of the computational geometry depending on the task at hand. meep_adjoint replaces all of these with the single new class Subregion.
